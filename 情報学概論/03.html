3. ソフトウェアエージェントの歴史

Alan Kay (1984) によるエージェントの起源。「アイデアは 1950 年代中期に John McCarthy が発案、用語は Oliver Selfridge が命名。ゴールが与えられると適切なコンピュータ操作を行い、行き詰まったときに人間にアドバイスを求められるシステムを念頭に置いていた」。
Pattie Maes（MIT, 1997）のインターフェースエージェント

「ユーザとアプリの間に挟まる層」ではなく パーソナルアシスタントとしてユーザの隣に並ぶ。ユーザはエージェントを飛び越えて直接操作もできる。
エージェントの学習方法：① 観察と模倣、② フィードバック適応、③ 例示によるトレーニング、④ 他エージェントへの質問。

4. 単体 AI vs マルチエージェントシステム

説明たとえ
単体 AI一人の人間の知性のモデル天才一人
マルチエージェントシステム社会の知性のモデルチームワーク

三人寄れば文殊の知恵。各エージェントは自分の目標を持ち、環境を知覚し、他エージェントと通信し、自律行動する。

5. LLM ベースのエージェント研究事例

BabyAGI：タスク駆動型自律エージェント

GPT-4 を核に「目標 → Task Creation → Task Queue → 優先順位付け → Execution → Memory → 次タスクへ」のサイクルを自律的に回す。原典。

AutoGen（Microsoft, 2023）

複数エージェントが 会話しながら タスクを解くフレームワーク。Joint chat / Hierarchical chat 等の柔軟な会話パターン。「Meta と Tesla の株価チャートを描いて」→ コードを書く AI とレビューする AI がペアでエラーを自動修正しながら完成。

LLM エージェントの社会的協調（Zhang et al., 2023）

チェスの駒移動の正誤判定で、3 つのエージェントが最初は誤答 → 互いの回答を参照して 反省と討論 を繰り返す → 3 回後に全員が正解「e2」に収束。社会心理学的な集団知性 が LLM にも現れる。

Theory of Mind（Huao Li et al., EMNLP 2023）

3 体（Alpha, Bravo, Charlie）が爆弾解除ゲームで協力。各エージェントが 信念状態 を更新しながらメッセージと行動を選択：「アルファが Room 0 を調べている → ブラボーを Room 3、チャーリーを Room 5 へ」。

Generative Agents（Park et al., UIST 2023）

25 体の LLM エージェントが Sims 風マップで生活。朝のルーティン、カフェでの会話「Hey Klaus, mind if I join you for coffee?」、市長選挙の話「Hey, have you heard anything about the upcoming mayoral election?」。リアルな人間行動を模倣できることを実証。

6. ゲーム理論とナッシュ均衡

複数の意思決定者が互いに影響し合う状況を 数学的に分析 する学問。ノーベル経済学賞を多数輩出（2007 Hurwicz・Maskin・Myerson、2012 Roth・Shapley 等）。

ナッシュ均衡の定義。戦略の組 $s^* = (s_1^*, \ldots, s_n^*)$ について、すべてのプレイヤー $i$ で $$ f_i(s^*) \geq f_i(s_i,\, s^*_{-i}),\quad \forall s_i \in S_i $$ が成り立つ状態。他人の戦略を固定したとき、自分の戦略を変えても得をしない ＝ 誰も抜け駆けの動機をもたない安定点。
具体例：Alex vs Bob のペイオフ行列
Bob: ABob: BBob: C
Alex: A7, 70, 105, 4
Alex: B0, 35, 5 ★4, 4
Alex: C2, 54, 23, 7

最良応答法：

  • Alex の最良応答：Bob=A → A、Bob=B → B、Bob=C → A
  • Bob の最良応答：Alex=A → B、Alex=B → B、Alex=C → C

両者の印が重なるのは (B, B) = (5, 5)。ここが ナッシュ均衡。

7. メカニズムデザイン：オークション

第1価格オークション：最高入札者が 自分の入札額 をそのまま払う。
第2価格オークション（ヴィックリー）：最高入札者が 2 番目の入札額 を払う。

なぜ第1価格は「不安定」なのか

第1価格で真の価値 $v$ で入札して落札すると利得 $u = v - v = 0$。利益ゼロなので必ず 入札を下げる（bid shading） 動機がはたらく。ではいくら下げるか？ → 他者の戦略次第で変わる（ベイズ・ナッシュ均衡）。$n$ 人・一様分布の場合の理論的最適：

$$ b^*(v) = \frac{n - 1}{n}\cdot v $$

つまり 2 人なら真の価値の半分。ただし相手の分布が分からないと計算できないため均衡が不安定。

第2価格で「正直入札」が支配戦略な直感
  • 真の価値 $v$ より 高く 入札 → 本来落札すべきでない場面でも落札してしまい 損
  • 真の価値より 低く 入札 → 本来落札できた場面を 逃す

どちらにずらしても得しないので $b = v$ が最善。これが 耐戦略性（strategyproof / incentive compatible）。

RegretNet：機械学習でオークションを自動設計（Dütting et al., ICML 2019）

複数アイテム × 複数バイヤーでは最適オークションを手で設計するのが極めて困難。これを ニューラルネットで学習 させる。

  • Allocation Network $g$：各バイヤーへの配分確率
  • Payment Network $p$：各バイヤーへの支払額
  • 損失：収益を最大化しつつ 事後後悔（嘘の申告で得られる追加利得）を最小化
$$ \mathrm{rgt}_i(w) = \mathbf{E}\!\left[\max_{v'_i} u^w_i(v_i; (v'_i, v_{-i})) - u^w_i(v_i; (v_i, v_{-i}))\right] $$

3 bidders × 10 items で収益 5.541（Myerson 5.310 を上回る）かつ後悔 < 0.002。

8. セキュリティゲームと拡張形ゲーム

ゲーム理論で守備側（Defender）が先に 混合戦略 を決め、攻撃者がそれを観察して最善攻撃を選ぶ Stackelberg ゲーム として定式化。実世界で広く運用：

システム用途
ARMORLAX 国際空港警察
PAWS野生動物保護
PROTECT / ARMOR-FISH米国沿岸警備隊
IRIS連邦航空保安官サービス
GUARDS米国運輸保安局（TSA）
TRUSTSロサンゼルス郡保安官事務所

（参照：Milind Tambe, Security and Game Theory, Cambridge UP, 2011）

ターミナル警備ゲームの例（2×2）
T1 を守るT2 を守る
T1 を攻撃5, −3−1, 1
T2 を攻撃−5, 52, −1

数値：(攻撃者の得点, 守備者の得点)。場所・時間ごとに異なるゲームを 拡張形ゲーム（ツリー構造）で表現する。

ポーカー AI：不完全情報ゲームの最前線

ポーカーは 相手の手札が見えない 不完全情報ゲーム。通常のゲーム理論を適用しにくい。CMU が Counterfactual Regret Minimization (CFR) でブレイクスルー：

システム達成
Libratus (2017)ヘッドアップ・ノーリミットでトッププロ 4 人に 20 日間で勝利（Science）
Pluribus (2019)6 人ポーカーで人間プロを超える。Science Breakthrough of the Year 次点

CFR の直感：「あの時別の手を打っていたらどうなったか（反事実的後悔）」を繰り返し計算し、後悔を最小化する戦略に収束させる手法。

講義のまとめ

  • AI は入れ子構造：AI ⊃ ML ⊃ DL ⊃ LLM。LLM はエージェントの「脳」
  • マルチエージェントは 社会の知性 モデル。BabyAGI・AutoGen・Generative Agents 等が研究最前線
  • ナッシュ均衡：他人の戦略を固定したとき自分も変える動機がない点。最良応答法で探す
  • ヴィックリーオークションは 正直入札が支配戦略＝耐戦略性が成り立つ。第1価格は不安定
  • RegretNet：ニューラルネットでオークションルール自体を学習する現代的アプローチ
  • セキュリティゲームは実世界で多数稼働。ポーカー AI は CFR で人間プロ超え
