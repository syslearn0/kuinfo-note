第4回 コンピュータの仕組み

講義

大きな絵：CPU と メモリの 2 つだけ

これまで「ビット → 数値表現 → 論理回路（組合せ・順序）」と部品を積み上げてきた。今回はそれらを組み合わせて コンピュータ全体 をつくる。本質は驚くほどシンプル：

主要部品の役割

部品役割具体例
演算装置計算・論理演算加減乗除、シフト
記憶装置データを保持レジスタ、メモリ、キャッシュ
データ転送装置データを送るセレクタ、ラッチ、バス
制御装置全体を制御するカウンタ、デコーダ、クロック
入出力装置外部とやりとりキーボード、画面、通信
2次記憶装置電源を切っても残るHDD、フラッシュメモリ

ワード長

一度に扱えるビット数を ワード長 といい、レジスタ幅・データバス幅・メモリ読書きの単位を決める。4 → 8 → 16 → 32 → 64 ビットと進化してきた。

記憶装置の階層構造

フリップフロップ（FF）：1 ビット記憶

NAND ゲート 2 つの交差接続で 1 ビットを保持できる。これが記憶装置の最小単位。

D-FF（クロック付き）：1 ビットレジスタ

クロック信号 CK の立ち上がりで入力 D を取り込み、Q として出力する。クロックはコンピュータ全体の「テンポ」。

n ビットレジスタ：D-FF を n 個並べる

メモリ：アドレスでアクセス

多数のレジスタを並べ、アドレス（番地）でひとつを選んで読み書きする。マンションの部屋番号で出し入れするイメージ。バイト／ワード単位で並列に出力できる。

データ転送回路

セレクタ（マルチプレクサ）

複数入力から 1 つを選ぶ。選択信号 $S = 0$ なら $D_0$、$S = 1$ なら $D_1$ を出力。$n$ ビットセレクタはこれを $n$ 本束ねたもの。

トランスファゲート・ラッチ

制御信号 $E$ で「通す／遮断」を切り替えるスイッチ。ラッチは「$E = 1$ で入力をそのまま出力、$E = 0$ で直前の値を保持」する。

バス：1 束の信号線を共用

装置ごとに配線すると本数が爆発するので、1 束の バス を全装置で共用し、各装置はトランスファゲートで接続する。同時に出力できるのは 1 装置だけ（同時出力すると異常電流で破損）。「複数人で 1 本のマイクを共有」のルールと同じ。

CPU の構成

要素略役割
演算回路ALU算術・論理・シフト

命令レジスタIR実行中の命令を一時保持

動作の核心：フェッチ → デコード → 実行

コンピュータは この 3 ステップを延々と繰り返す だけ。これを知るとブラックボックスが透明になる。

命令の種類

分岐命令が「プログラムらしさ」の核心。PC を書き換える命令があるだけで、ループ・関数呼び出し・条件分岐のすべてが実現できる。

プログラム例①：メモリの指定範囲のデータ総和

外部から「開始アドレス」と「個数」を受け取り、その範囲の合計を出力する。

0:  IN  0       外部から開始アドレスを GR0 に
1:  IN  1       外部から個数を GR1 に
2:  CLR 2       GR2 ← 0（合計用）
3:  LD  3 0     GR3 ← Mem[GR0]
4:  ADD 2 3     GR2 ← GR2 + GR3
5:  INC 0       GR0 を 1 進める（次のアドレス）
6:  DEC 1       GR1 を 1 減らす（残り個数）
7:  BP          直前の結果が正なら
8:     3        → 3 番地に戻る（ループ）
9:  OUT 2       合計を出力
10: HALT        停止
動作トレース：開始 = 11 番地、個数 = 3、データ = 5, 4, 7
  • 初期：GR0 = 11, GR1 = 3, GR2 = 0
  • 1 周目：GR3 ← 5、GR2 = 5、GR0 = 12、GR1 = 2 → 正 → 戻る
  • 2 周目：GR3 ← 4、GR2 = 9、GR0 = 13、GR1 = 1 → 正 → 戻る
  • 3 周目：GR3 ← 7、GR2 = 16、GR0 = 14、GR1 = 0 → 0 → ループ終了
  • 出力：16

プログラム例②：1 から N までの和

0: IN  0        GR0 ← N
1: CLR 1        GR1 ← 0（合計用）
2: ADD 1 0      GR1 ← GR1 + GR0
3: DEC 0        GR0 を 1 減らす
4: BP           GR0 が正なら
5:    2         → 2 番地に戻る
6: OUT 1        GR1 を出力
7: HALT
なぜこれで $1 + 2 + \cdots + N$ になるか

$N$ から 1 ずつ減らしながら毎回 GR1 に加算する：$N + (N-1) + \cdots + 1 = 1 + 2 + \cdots + N$。
$N = 3$ なら：GR1 ← 3, GR0 = 2 → GR1 ← 5, GR0 = 1 → GR1 ← 6, GR0 = 0 で終了、出力 6。

機械語 → アセンブリ言語 → 高級言語

CPU が直接理解できるのは 機械語（0/1 の列） だけ。それを人間が書きやすいレベルへと 翻訳の階層 で持ち上げる：

講義のまとめ

  • コンピュータの本体は CPU と メモリ の 2 つ。命令もデータもすべてメモリ上の 0/1 の列
  • FF → レジスタ → メモリ、AND/OR/NOT → 全加算器 → ALU、と部品が階層的に組み上がる
  • CPU は フェッチ → デコード → 実行 の 3 ステップを繰り返すだけ
  • 命令は 5 カテゴリ（演算・分岐・ロード／ストア・入出力・制御）。分岐がループや条件分岐の源
  • 0/1 の機械語を人間が書きやすくしたのがアセンブリ／高級言語、それを翻訳するのがアセンブラ／コンパイラ
