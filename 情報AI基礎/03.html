第3回 ハミング符号と論理回路

講義

大きな絵

前半は パリティ／ハミング符号（第2回の続き）で、誤りをどう「検出」し「位置特定」するか。後半は 論理関数・論理回路。これまで「ビット列でデータを表す」話だったが、ここから「ビットを 処理する」回路の世界に踏み込む。AND・OR・NOT の 3 つの部品だけで、原理的にどんな計算もできる。

パリティ検査符号

データ $k$ ビットに パリティ検査ビット 1 つを足し、全体の 1 の個数を 偶数 に揃える。受信側で 1 の個数を数えて奇数なら誤りあり。

受信ビット列1 の個数判定
0110 0（正常）2（偶）OK
0010 0（1 ビット誤り）1（奇）誤り検出
0110 1（検査ビット誤り）3（奇）誤り検出
0011 0（2 ビット同時誤り）2（偶）検出不可
1 ビットしか検出できず、2 ビット同時に化けると見逃す。

ハミング符号（7,4）の構成

情報 4 ビットに 3 つの検査ビット を付け、計 7 ビットにする。各検査ビットは 異なる情報ビットの組合せ のパリティ：

例：$[0110]$ を符号化

  • 検査①（d1, d2, d3 = 0, 1, 1）→ 1 が 2 個（偶）→ p1 = 0
  • 検査②（d2, d3, d4 = 1, 1, 0）→ 1 が 2 個（偶）→ p2 = 0
  • 検査③（d1, d2, d4 = 0, 1, 0）→ 1 が 1 個（奇）→ p3 = 1

符号語：0110 001

ハミング符号による誤り訂正

受信側で 3 つの「拡張パリティ検査」を再計算する。各検査の値を並べた 3 ビットを シンドローム と呼ぶ。

核心。「誤り位置」と「シンドローム」が 一対一対応 するよう検査範囲を設計してある。受信側はシンドロームを見るだけで、どのビットが化けたかを特定し、反転すれば直る。
シンドローム [C1 C2 C3]エラー位置
000エラーなし
101d1（1 ビット目）
111d2（2 ビット目）
110d3（3 ビット目）
011d4（4 ビット目）
100p1（5 ビット目）
010p2（6 ビット目）
001p3（7 ビット目）
なぜ位置が特定できるのか（設計の直感）

3 ビットのシンドロームには $2^3 = 8$ 通りのパターンがある。「エラーなし（1 通り）」＋「7 ビット中どこか 1 ビット誤り（7 通り）」＝ ちょうど 8 通り。これに ぴったり一対一 対応するように、各検査の担当範囲が選ばれている。

例：$[0110\,001]$ の 1 ビット目が化けて $[0010\,001]$ になった場合

3 つの検査を計算するとシンドロームは 101、表より 1 ビット目が誤り。反転して復元 → $[0110\,001]$。

能力の限界

誤りビット数「誤りあり」と気付ける？「どこか」まで特定？
0——（正常）
1○ 検出○ 訂正可能
2○ 検出（シンドローム ≠ 000）× 誤訂正してしまう
3 以上×（検出できない場合あり）×

つまり 検出だけなら 2 ビットまで、訂正なら 1 ビットまで。場面に応じて使い分ける：

場面必要な能力
インターネット（TCP/IP）検出のみ（再送できる）→ CRC
HDD・CD・ECC メモリ訂正（再送できない）→ ハミング系
宇宙探査機・放送訂正（往復遅延／一方向）→ より強力な符号

論理変数と論理関数

論理変数：0 か 1 のどちらかをとる変数。
論理関数：複数の論理変数を入力して 0/1 を出力する関数：

$$ f: \{0, 1\}^n \to \{0, 1\} $$

コンピュータの計算は すべて 論理関数として表現できる。

例：1 桁の 2 進加算（全加算器）

入力 3 ビット $x_i, y_i, c_i$（加数・被加数・下からの桁上げ）、出力 2 ビット $s_i, c_{i+1}$（和・上への桁上げ）。

基本論理演算（AND・OR・NOT）

任意の論理関数は AND・OR・NOT の 3 つだけ で組み立てられる（関数完全性）。

加算の論理式表現

真理値表で「出力が 1 になる入力パターン」を AND で表し、それらを OR でつなぐ（主加法標準形）：

$$ s_i = \bar{x}_i\bar{y}_i c_i + \bar{x}_i y_i \bar{c}_i + x_i\bar{y}_i\bar{c}_i + x_i y_i c_i $$ $$ c_{i+1} = x_i y_i + x_i c_i + y_i c_i $$

この式から直接、AND・OR・NOT ゲートで回路を組める。

2 列が完全一致 → 同値。直感的には「A または B でない ＝ A でないかつ B でない」。

コンセンサスの法則：$xy + \bar{x}z + yz = xy + \bar{x}z$

$yz$ という項は冗長：$x = 1$ なら $yz$ が成り立つときは $xy$ も成り立つ。$x = 0$ なら $yz$ が成り立つときは $\bar{x}z$ も成り立つ。どちらの場合も $yz$ は他に吸収される。
用途：論理式の簡略化＝ゲート数の削減（回路設計の最適化）。

命題論理とプログラムの条件式

C 言語の論理演算子は、ブール代数の演算と 同じもの です：

数学C 言語意味
$\land$&&AND（両方）
$\lor$||OR（どちらか）
$\lnot$!NOT（反転）

つまり if 文の条件式はそのままブール代数の計算で、回路に翻訳できる。

論理ゲートと CMOS

論理演算を電子回路で実現したのが 論理ゲート。現代の半導体は CMOS（相補型 MOS）方式で、PMOS と NMOS のトランジスタを組み合わせる：

  • NOT ゲート：トランジスタ 2 個
  • 2 入力 NAND：トランジスタ 4 個

CPU の中身は、究極的にはこの 数十億個 のトランジスタからできている。

組合せ回路：全加算器

論理ゲートを 記憶を持たず 配線でつないだ回路を 組合せ回路 と呼ぶ。出力は現在の入力だけで決まる。先ほどの式を素直に回路化する：

順次桁上げ加算器（n 桁）

全加算器を n 個直列 につなぐと n ビット加算器ができる。下の桁の桁上げが確定してから次の桁が計算できる：

桁上げ選択加算器（高速化アルゴリズム）

「桁上げ = 0 のとき」「桁上げ = 1 のとき」の 両方を並列に計算 しておき、前段から桁上げが届いたら正しい方を 選ぶ。これで計算時間が約半分に。ハードウェアの並列性を活かした計算法＝ ハードウェアアルゴリズム。

順序回路（記憶を持つ回路）

種類出力を決める要因
組合せ回路現在の入力だけ
順序回路現在の入力 ＋ 内部状態（過去の記憶）

例：自動販売機。内部状態 = 「これまでに投入された硬貨の合計」。新しい硬貨で状態が更新され、合計が値段に達したら商品を出す。これが 有限オートマトン の最小例で、CPU の制御部分の基礎。次回は実際に CPU を組み立てる。

講義のまとめ

  • パリティ検査は 1 ビット誤りの 検出、ハミング符号は 1 ビット誤りの 訂正
  • シンドロームと誤り位置を 一対一対応 させるのが訂正符号の核心
  • 論理関数 $\{0,1\}^n \to \{0,1\}$ が計算の数学的単位。真理値表で定義できる
  • AND・OR・NOT の 3 つで 任意の論理関数 が組める（関数完全性）
  • ブール代数の法則で式を変形 → ゲート数を削減できる
  • 組合せ回路（記憶なし）＋順序回路（記憶あり）＝ コンピュータの部品箱
